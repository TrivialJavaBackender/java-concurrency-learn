# Concurrency Interview Prep

## Проект
Этот проект — площадка для практики Java Concurrency (Kotlin + Java concurrency APIs).

## Команды

### Прогресс
Когда пользователь говорит "прогресс", "статус" или "как дела" — читай PROGRESS.md и покажи текущий статус.

### Проверка упражнения
Когда пользователь говорит "проверь Ex01" (или любой номер):
1. Прочитай соответствующий файл из src/main/kotlin/exercises/
2. Проверь что TODO заменены на реализацию
3. Попробуй скомпилировать: `mvn compile -q`
4. Если компилируется — запусти: `mvn exec:java -Dexec.mainClass="exercises.ExXX_NameKt" -q`
5. Проверь корректность вывода
6. Проведи детальный code review:
   - Найди все race condition, неправильное использование локов, утечки, неверную логику wait/notify
   - Укажи на субоптимальные решения и объясни почему они плохи
   - Проверь соответствие условию задачи (все требования выполнены?)
   - Не пропускай замечания — лучше написать лишнее, чем упустить баг
7. Помечай упражнение выполненным (✅ в PROGRESS.md) ТОЛЬКО если нет серьёзных замечаний.
   Если есть — оставь ⬜, перечисли что нужно исправить, и жди повторной проверки.

### Следующий модуль
Когда пользователь говорит "следующий" или "next":
1. Найди первый незавершённый модуль в PROGRESS.md
2. Прочитай соответствующий файл теории из theory/ (см. таблицу в ROADMAP.md)
3. Покажи краткое содержание теории и ключевые вопросы для самопроверки
4. Предложи начать с первого невыполненного упражнения модуля

### Квиз
Когда пользователь говорит "квиз" или "quiz" — задай 5 случайных вопросов из INTERVIEW_QUESTIONS.md, жди ответа, оцени.

## Упражнения
Когда пользователь просит добавить или обновить какое-то из упражнений, новое упражнение должно быть
достаточно challenging, и не содержать прямых указаний того, что надо сделать в коде.

Никогда не давай подсказки по реализации упражнений, если пользователь явно не попросил.
Когда пользователь говорит "начать" — просто сообщи что файл готов к реализации и жди.

## Сборка
- `mvn compile` — компиляция
- `mvn exec:java -Dexec.mainClass="exercises.Ex01_ThreadBasicsKt"` — запуск упражнения

- theory/SYNCHRONIZERS.md — Модуль 6
