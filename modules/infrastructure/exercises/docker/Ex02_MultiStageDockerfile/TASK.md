# Ex02: Multi-stage Dockerfile

**Модуль:** 1 — Docker
**Сложность:** ★★★☆☆
**Тема:** Multi-stage build, оптимизация размера образа, кэширование слоёв

## Контекст

Разработчики жалуются, что Docker образ весит 600 МБ — это много для хранения в registry
и медленно деплоится. Причина: в образ попадает Maven, JDK и весь кэш `.m2`.
При этом для запуска приложения нужен только JRE и собранный `jar`.

## Задача

Переписать `Dockerfile` с использованием многоэтапной сборки (multi-stage build).
Финальный образ должен содержать только необходимое для запуска.

Требования:
- Первый stage: компиляция через Maven (не копируй `target/` снаружи — собирай внутри)
- Второй stage: минимальный JRE образ, только собранный jar
- Зависимости Maven должны кэшироваться отдельно от исходного кода
- Финальный образ — non-root, с HEALTHCHECK
- Размер финального образа не должен превышать 150 МБ

## Файлы для изменения

- `Dockerfile` — написать с нуля (два stage)

## Проверка

Упражнение считается выполненным, когда:
- [ ] В Dockerfile есть минимум два `FROM` (два stage)
- [ ] `pom.xml` копируется до `src/` (кэширование зависимостей)
- [ ] Финальный образ не содержит Maven и JDK (только JRE)
- [ ] `COPY --from=<stage>` используется для переноса jar
- [ ] Non-root USER и HEALTHCHECK присутствуют
- [ ] `docker buildx build --dry-run` проходит без ошибок
