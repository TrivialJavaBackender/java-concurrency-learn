# Ex06: Оптимизация Dockerfile

**Модуль:** 1 — Docker
**Сложность:** ★★★★☆
**Тема:** Layer caching, размер образа, .dockerignore

## Контекст

Команда написала Dockerfile, который работает, но имеет проблемы с производительностью
сборки и размером образа. CI/CD пайплайн тратит 8 минут на пересборку при каждом
изменении в коде, хотя зависимости не менялись.

Текущий Dockerfile (`Dockerfile.bad`) предоставлен для анализа.

## Задача

Проанализировать `Dockerfile.bad`, найти все проблемы и написать оптимальный `Dockerfile`.

Требования:
- При изменении только исходного кода (не `pom.xml`) зависимости берутся из кэша
- Итоговый образ меньше, чем у `Dockerfile.bad` (можно проверить через `docker history`)
- Нет лишних файлов в образе (`.git`, `target/`, IDE файлы)
- Объединить связанные RUN команды в одну
- Убрать все антипаттерны из `Dockerfile.bad`

## Файлы для изменения

- `Dockerfile` — написать оптимальную версию
- `.dockerignore` — создать

## Проверка

Упражнение считается выполненным, когда:
- [ ] В `.dockerignore` исключены: `target/`, `.git`, `.idea`, `*.md`
- [ ] `pom.xml` копируется до `src/` и зависимости кэшируются отдельно
- [ ] Нет `ADD` — только `COPY`
- [ ] Нет `:latest` тега базового образа
- [ ] Non-root USER
- [ ] `docker buildx build --dry-run` проходит без ошибок
