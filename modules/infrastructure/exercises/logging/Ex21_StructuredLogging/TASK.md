# Ex21: Структурированное логирование

**Модуль:** 5 — Logging
**Сложность:** ★★★☆☆
**Тема:** MDC, бизнес-контекст в логах, уровни логирования

## Контекст

`OrderService` логирует только строковые сообщения без контекста. При расследовании
проблем невозможно понять "для какого пользователя", "какой заказ", "на какую сумму".
Нужно добавить структурированный контекст через MDC.

## Задача

Реализовать `OrderService` с правильным структурированным логированием.

Требования:
- При создании заказа: добавить в MDC `userId`, `orderId`, `orderAmount`
- Логировать начало обработки на INFO, детали на DEBUG
- При ошибке валидации: WARN (ожидаемая ситуация)
- При неожиданной ошибке: ERROR с полным stacktrace (второй аргумент)
- MDC очищается после обработки (в finally)
- Нет `log.error("Error: " + e.getMessage())` — только `log.error("message", e)`
- Нет конкатенации строк в log вызовах — только параметры `{}`

## Файлы для изменения

- `OrderService.java` — написать с нуля

## Рабочий процесс

Реализуй `OrderService.java` в этой директории, затем скопируй в проект и скомпилируй:
```bash
cp exercises/logging/Ex21_StructuredLogging/OrderService.java src/main/java/by/pavel/logging/
mvn compile -q
```

## Проверка

Упражнение считается выполненным, когда:
- [ ] `mvn compile -q` проходит без ошибок
- [ ] `MDC.put()` вызывается для userId, orderId, orderAmount
- [ ] `MDC.clear()` или `MDC.remove()` в finally блоке
- [ ] Нет конкатенации строк в log методах
- [ ] ERROR уровень используется только для неожиданных исключений с stacktrace
- [ ] WARN уровень для ожидаемых проблем (валидация)
